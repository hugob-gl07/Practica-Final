================================================================================
COMPLEJIDADES CRÍTICAS
================================================================================

MatrizHabitacion.getCelda(fila, columna)
  └─ O(fila + columna) = O(n+m)
  └─ Búsqueda secuencial en LSE
  └─ Aceptable para matrices pequeñas (10x10 a 20x20)

MatrizHabitacion.obtenerCeldasAlcanzables(fila, col, rango)
  └─ O(n*m) en peor caso
  └─ BFS desde posición con rango
  └─ Típicamente mucho menor que n*m en la práctica

GrafoHabitaciones.caminoMinimoHabitaciones(origen, destino)
  └─ O(V+E) del BFS
  └─ V ≈ 5-20 habitaciones, E ≈ 2-4 puertas por habitación
  └─ Muy eficiente para grafos dispersos

GrafoHabitaciones.dijkstraHabitaciones(origen, destino)
  └─ O((V+E)logV) del Dijkstra
  └─ Óptimo para caminos ponderados

================================================================================
NOTAS DE IMPLEMENTACIÓN
================================================================================

MatrizHabitacion:
  • Implementar con LSE<LSE<Celda>> para cumplir restricción
  • Método getCelda() debe iterar hasta encontrar la fila y columna
  • Considerar cache de fila actual para optimización futura
  • obtenerCeldasAlcanzables() debe usar BFS con distancia Manhattan

Habitacion:
  • Encapsula matriz + metadatos
  • obtenerPuerta(destino) busca en LSE de puertas
  • obtenerEnemigos() filtra celdas con Entidad != null
  • obtenerObjetos() filtra celdas con Objeto != null

GrafoHabitaciones:
  • Usar composición: private Grafo grafo;
  • NO extender Grafo, usar composición
  • Métodos de camino delegan a grafo.caminoMinimo() y grafo.dijkstra()
  • Convertir LSE<Nodo> a LSE<Habitacion> en métodos de camino
  • Persistencia: usar formato JSON con tripletas RDF
